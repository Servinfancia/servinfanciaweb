Guía AppSheet para Servinfancia SAS

¡Bienvenido/a! Esta guía te ayudará a crear tu primera aplicación en AppSheet para Servinfancia SAS, paso a paso. ¡Imagina que es como construir con bloques digitales!

Glosario: Entendiendo el Mundo AppSheet

Aquí tienes un diccionario con palabras raras que usaremos. ¡No te asustes! Son más fáciles de lo que parecen.

  • AppSheet: Es una plataforma (como un taller de herramientas online) que te deja crear apps para celulares, tablets y computadoras sin necesidad de escribir código complicado. Es como si te dieran piezas de LEGO y tú las unes para hacer lo que quieras.
  • Google Sheets: Piensa en esto como tu cuaderno digital online, ¡pero más poderoso que Excel para trabajar en equipo! Aquí guardarás toda la información (datos) que tu app de AppSheet usará y mostrará. Es como la base de datos de tu app, pero en un formato de hojas de cálculo que ya conoces.
  • UX (User Experience - Experiencia de Usuario): Suena técnico, ¿verdad? Pero solo significa qué tan fácil y agradable es para una persona usar tu aplicación. Una buena UX es como un juego bien diseñado: sabes dónde hacer clic y te diviertes usándolo. Queremos que usar tu app de Servinfancia sea ¡pan comido!
  • Views (Vistas): Son las diferentes pantallas que tendrá tu app. Cada vista muestra la información de tus Google Sheets de una manera especial. Es como tener diferentes formas de ver tus cartas: todas juntas, en un calendario, en un mapa, etc. AppSheet tiene varios tipos:
    • Calendar (Calendario): Muestra tus datos como en un calendario. Perfecto para ver fechas de entrega de alimentos o programación de rutas. Ejemplo: Ver en qué días hay entregas programadas.
    • Deck (Mazo): Muestra la información como una serie de tarjetas resumidas, una debajo de la otra. Ideal para una vista rápida de pedidos o niños. Ejemplo: Una lista de niños con su nombre y grupo etario.
    • Table (Tabla): Muestra los datos como una tabla, similar a como se ven en Google Sheets, con filas y columnas. Útil para ver mucha información detallada a la vez. Ejemplo: Una tabla con todos los detalles de los pedidos.
    • Gallery (Galería): Muestra la información como una serie de tarjetas visuales, usualmente con imágenes. Si tuvieras fotos de los puntos de entrega, podrías usarla. Ejemplo: Mostrar fotos de los kits de alimentos (si las tuvieras).
    • Detail (Detalle): Muestra toda la información de UN solo elemento (una fila de tu tabla). Por ejemplo, al hacer clic en un niño en una vista de "Deck", se abriría una vista de "Detail" con toda su información.
    • Map (Mapa): Si tienes direcciones, esta vista las muestra en un mapa. ¡Genial para planificar las rutas de entrega! Ejemplo: Ver en un mapa dónde están todos los puntos de entrega.
    • Chart (Gráfico): Muestra tus datos en forma de gráficos (barras, pastel, líneas). Muy útil para entender tendencias o resumir información. Ejemplo: Un gráfico de cuántos niños hay por grupo etario.
    • Dashboard (Tablero): Es una pantalla especial que te permite mostrar varias vistas diferentes al mismo tiempo. Como el tablero de un carro, que te muestra la velocidad, la gasolina, etc., todo en un solo lugar. Ejemplo: Mostrar un mapa de rutas y una tabla de pedidos pendientes en la misma pantalla.
    • Form (Formulario): Es la pantalla que usas para agregar nueva información o editar la existente. Como cuando llenas un formulario online para registrarte en algo. Ejemplo: Un formulario para registrar un nuevo niño o un nuevo pedido.
    • Card (Tarjeta): Es un tipo de vista más nuevo y flexible que te permite diseñar cómo se muestra cada ítem de tus datos de forma muy personalizada, como si crearas pequeñas "tarjetas de presentación" para cada dato. Ejemplo: Tarjetas personalizadas para cada ruta mostrando el conductor, vehículo y número de entregas.
  • Columns (Columnas): En tus Google Sheets (y por lo tanto en AppSheet), cada hoja tiene columnas. Cada columna guarda un tipo específico de información. Por ejemplo, en una hoja de "Niños", podrías tener columnas para "Nombre", "Fecha de Nacimiento", "Grupo Etario". En AppSheet, defines qué tipo de dato va en cada columna (texto, número, fecha, imagen, etc.).
  • Design View (Vista de Diseño): Es el "taller" dentro de AppSheet donde construyes y personalizas tu app. Aquí eliges las vistas, configuras las columnas, creas acciones, etc. Es donde ocurre toda la magia.
  • Fórmulas: Son como las fórmulas que usas en Excel, pero adaptadas a AppSheet. Te permiten hacer cálculos, tomar decisiones lógicas (si pasa esto, haz aquello), o mostrar información específica. ¡Son súper poderosas!
    • NOW(): Es una fórmula que te da la fecha y hora EXACTA de este momento. Útil para saber cuándo se registró un pedido o cuándo se hizo una entrega. Ejemplo: En un formulario de entrega, puedes tener una columna que automáticamente se llena con `NOW()` cuando se marca como entregado.
    • USEREMAIL(): Esta fórmula te da el correo electrónico de la persona que está usando la app en ese momento (si inició sesión con Google). Sirve para saber quién hizo qué. Ejemplo: Registrar qué usuario marcó un pedido como "alistado".
    • LOOKUP(): Esta es como un detective. Le das un valor, le dices en qué tabla y columna buscar, y qué columna te tiene que devolver como resultado. Ejemplo: Si tienes el ID de un niño en la tabla de "Pedidos", puedes usar `LOOKUP()` para encontrar el nombre completo de ese niño en la tabla de "Niños". `LOOKUP([ID Niño], "Niños", "ID Niño", "Nombre Completo")` buscaría el `ID Niño` en la tabla "Niños" y devolvería el "Nombre Completo".
  • Botones (Actions o Acciones): Son elementos en tu app en los que los usuarios pueden hacer clic para que algo suceda. Por ejemplo, un botón para "Guardar", "Marcar como Entregado", "Actualizar Grupo Etario".
  • Configuraciones (Settings): Son las opciones que ajustas en AppSheet para cambiar cómo se ve y funciona tu app. Desde el nombre de la app, el logo, hasta cosas más avanzadas.
  • Parámetros (Parameters): A veces, en configuraciones o fórmulas avanzadas, puedes usar parámetros, que son como variables que puedes ajustar para cambiar el comportamiento sin modificar toda la lógica. (No te preocupes mucho por esto al inicio).
  • Tablas (Tables): En AppSheet, cada hoja de tu Google Sheets se convierte en una "Tabla". Es la fuente de tus datos. Si tienes una hoja para "Niños" y otra para "Pedidos", tendrás dos tablas en AppSheet.
  • Slices (Rebanadas): Imagina que tienes una tabla enorme de datos, pero solo quieres mostrar una pequeña parte de ella en una vista específica. Un "Slice" es como cortar una "rebanada" de tu tabla que cumple ciertos criterios. Ejemplo: Si tienes una tabla con todos los pedidos (entregados y pendientes), podrías crear un Slice llamado "Pedidos Pendientes" que solo muestre los que aún no se han entregado.
  • Acciones (Actions): Son las tareas que tu app puede realizar, usualmente iniciadas por botones o eventos. Pueden ser cosas como agregar una nueva fila a una tabla, modificar datos existentes, navegar a otra vista, enviar un email, etc. Ejemplo: Una acción "Actualizar Estado a Entregado" que cambia el valor de una columna en la tabla de pedidos.
  • Eventos (Events): Son cosas que suceden en tu app (como guardar un formulario, sincronizar datos) que pueden disparar Acciones automáticamente. Ejemplo: Un evento podría ser que "Cuando se guarde un nuevo pedido", automáticamente se envíe una notificación.
  • Autenticación (Authentication): Es el proceso de verificar quién está usando la app. En nuestro caso, usaremos la autenticación con Google, lo que significa que los usuarios iniciarán sesión con su cuenta de Google. Esto ayuda a la seguridad y a saber quién hace qué.
  • Vistas Primarias (Primary Views): Son las pantallas principales de tu app, las que aparecen en el menú principal (usualmente abajo o en un menú lateral). Son los puntos de entrada a las diferentes secciones de tu app.
  • Vistas de Referencia (Reference Views o Ref Views): Son vistas que no aparecen en el menú principal, sino que se accede a ellas desde otras vistas. Por ejemplo, la vista de "Detalle" de un pedido es una vista de referencia, porque llegas a ella haciendo clic en un pedido desde una lista.
  • Sincronización (Sync): Es el proceso mágico que mantiene tus datos actualizados entre la app en el celular/tablet/computadora y tu Google Sheet. Cuando alguien hace un cambio en la app, se sincroniza para que se guarde en el Google Sheet. Y si cambias algo directamente en el Google Sheet, la app se sincroniza para mostrar esos cambios. AppSheet lo hace automáticamente, pero a veces hay un botón para forzar la sincronización.
  • Modo Offline (Offline Mode): AppSheet permite que tu app funcione incluso si no tienes internet en ese momento. Los usuarios pueden seguir viendo datos y agregando nueva información. Cuando la app vuelve a tener conexión, sincroniza todos los cambios. ¡Muy útil para entregas en zonas sin buena señal!
  • Permisos (Permissions): Definen qué pueden hacer los usuarios en tu app. ¿Pueden agregar datos? ¿Solo verlos? ¿Editar? ¿Borrar? Esto se configura en AppSheet, a menudo ligado a los roles.
  • Roles (Roles): Puedes definir diferentes tipos de usuarios (roles), como "Administrador" o "Repartidor". Cada rol puede tener diferentes permisos. Por ejemplo, un "Administrador" podría ver y editar todo, mientras que un "Repartidor" solo podría ver sus rutas y marcar pedidos como entregados. (Esto es más avanzado, pero bueno que lo sepas).
  • View Options (Opciones de Vista): Cada tipo de vista (Tabla, Deck, Calendario, etc.) tiene sus propias opciones para personalizarla:
    • Sort by (Ordenar por): Te permite elegir en qué columna y en qué orden (ascendente o descendente) se mostrarán los datos. Ejemplo: Ordenar los pedidos por fecha de entrega, del más nuevo al más viejo.
    • Group by (Agrupar por): Te permite agrupar filas que tienen el mismo valor en una columna específica. Ejemplo: Agrupar pedidos por "Estado" (Pendiente, En Ruta, Entregado).
    • Group aggregate (Agregado de grupo): Cuando agrupas datos, puedes hacer cálculos dentro de cada grupo, como contar cuántos ítems hay (Count), sumar valores (Sum), encontrar el promedio (Average), etc. Ejemplo: Si agrupas pedidos por "Estado", puedes mostrar cuántos pedidos hay en cada estado.
    • Use tabs in mobile view (Usar pestañas en vista móvil): Para vistas de Dashboard, esta opción permite que las diferentes vistas dentro del dashboard se muestren como pestañas en celulares, para ahorrar espacio.
    • Interactive mode (Modo Interactivo): Para vistas de Dashboard, si está activado, cuando seleccionas algo en una vista del dashboard, puede filtrar o afectar lo que se muestra en otras vistas del mismo dashboard. ¡Muy dinámico!

Plan de Acción: ¡Nuestro Mapa del Tesoro para la App!

Este es nuestro plan general. ¡No te preocupes si parece mucho, iremos paso a paso!

  1. Etapa 1: ¡Creando Nuestro Cuaderno Digital y la Puerta de Entrada!
    • Qué haremos: Vamos a crear el archivo de Google Sheets donde vivirá toda nuestra información. Luego, conectaremos este cuaderno a AppSheet y haremos la primera pantalla de bienvenida con inicio de sesión de Google. ¡Será como poner el letrero de "Bienvenidos a Servinfancia" en nuestra app!
  2. Etapa 2: ¡Organizando a los Peques! (Gestión de Grupos Etarios)
    • Qué haremos: Añadiremos una sección para registrar a los niños y niñas, y una forma de actualizar sus grupos etarios automáticamente o con un botón. Esto es como tu macro de Excel para grupos etarios, ¡pero ahora en la app!
  3. Etapa 3: ¡A Preparar los Pedidos! (Programación de Alistamiento)
    • Qué haremos: Crearemos una parte en la app para registrar los pedidos de alimentos, ver qué se necesita y marcar cuándo están listos para ser enviados. ¡Adiós a los enredos del archivo único!
  4. Etapa 4: ¡Rutas en Marcha! (Despacho de Rutas)
    • Qué haremos: Montaremos la funcionalidad para asignar pedidos a rutas, ver las rutas en un mapa (¡si tenemos direcciones!) y marcar cuándo se han entregado los alimentos. Los repartidores podrán actualizar el estado desde su celular.
  5. Etapa 5: ¡Poniéndola Bonita y Poderosa! (Mejoras y Extras)
    • Qué haremos: Una vez que lo básico funcione, podemos añadir gráficos para ver estadísticas, mejorar cómo se ve la app, configurar notificaciones (quizás un aviso cuando un pedido está listo), y definir permisos si diferentes personas necesitan ver cosas distintas.

Paso a Paso: ¡Construyendo Nuestra App Bloque por Bloque!

Etapa 1: ¡Creando Nuestro Cuaderno Digital y la Puerta de Entrada!

En esta primera etapa, sentaremos las bases: nuestro archivo de Google Sheets y la primera pantalla de nuestra app.

Paso 1.1: Crear el Libro de Google Sheets (Nuestro Cuaderno Digital)
  • Acción: Abre tu navegador web (Chrome, Firefox, etc.) y ve a sheets.google.com. Si no has iniciado sesión en Google, te pedirá que lo hagas.
  • Qué hace: Te lleva a la página principal de Google Sheets.
  • Para qué sirve: Desde aquí podemos crear nuestro nuevo "cuaderno".
  • Verificación: Verás una página con la opción de crear una "Hoja de cálculo en blanco" (suele ser un gran signo "+" de colores).
Paso 1.2: Iniciar una Nueva Hoja de Cálculo
  • Acción: Haz clic en "Hoja de cálculo en blanco" o el signo "+".
  • Qué hace: Crea un nuevo archivo de Google Sheets sin título.
  • Para qué sirve: Este será el archivo principal donde guardaremos TODOS los datos de nuestra app Servinfancia.
  • Verificación: Se abrirá una nueva hoja de cálculo vacía, con "Hoja de cálculo sin título" en la parte superior izquierda.
Paso 1.3: Ponerle Nombre a Nuestro Cuaderno Digital
  • Acción: Haz clic donde dice "Hoja de cálculo sin título" en la parte superior izquierda. Escribe un nombre descriptivo, por ejemplo: DatosAppServinfancia. Presiona Enter.
  • Qué hace: Le da un nombre a tu archivo de Google Sheets.
  • Para qué sirve: Para que sea fácil de encontrar y saber qué contiene.
  • Verificación: El nombre en la parte superior izquierda cambiará a DatosAppServinfancia.
Paso 1.4: Crear la Primera Hoja para la Bienvenida (Opcional, pero buena práctica)

Aunque AppSheet puede crear vistas sin una tabla de datos específica para una simple bienvenida, tener una hoja de control o configuración puede ser útil más adelante. Por ahora, solo la crearemos, pero no es estrictamente necesaria para una pantalla de bienvenida sin botones o datos dinámicos.

Podemos decidir más adelante si necesitamos datos específicos para la bienvenida. Por ahora, la pantalla de bienvenida será estática, configurada directamente en AppSheet.

  • Acción: En la parte inferior de la hoja de Google Sheets, verás una pestaña que dice "Hoja1". Haz clic derecho sobre ella (o en la flechita) y selecciona "Cambiar nombre". Escribe ConfiguracionApp y presiona Enter.
  • Qué hace: Renombra la primera hoja de cálculo.
  • Para qué sirve: Para tener un lugar donde poner ajustes generales de la app si los necesitamos más adelante. Por ahora, puede estar vacía o tener una columna simple como "Version" con el valor "1.0". (Esto es opcional).
  • Verificación: La pestaña inferior ahora dice ConfiguracionApp.
Ejemplo de la hoja ConfiguracionApp (Opcional por ahora):
Ajuste (Columna A) Valor (Columna B)
MensajeBienvenida Seleccione una opción para gestionar pedidos
VersionApp 1.0

Nota: Para la bienvenida que nos piden (solo título y mensaje, sin botones), AppSheet puede mostrar esto sin necesidad de leerlo desde esta tabla. Pero tenerla es una buena práctica para textos que podrías querer cambiar después sin tocar la estructura de la app. Por ahora, la crearemos pero no la usaremos para la bienvenida inicial.

Paso 1.5: ¡Hora de Conocer AppSheet! Ir a AppSheet.com
  • Acción: Abre una nueva pestaña en tu navegador y ve a www.appsheet.com.
  • Qué hace: Te lleva a la página principal de AppSheet.
  • Para qué sirve: Desde aquí crearemos nuestra nueva aplicación.
  • Verificación: Verás la página de inicio de AppSheet. Busca un botón que diga "Start for free" o "Crear una aplicación".
Paso 1.6: Iniciar la Creación de la App y Conectar con Google Sheets
  • Acción: Haz clic en "Start for free" o similar. AppSheet te pedirá que inicies sesión con tu proveedor de datos (Google, Microsoft, etc.). Elige "Google" (o la cuenta con la que creaste el Google Sheet).
  • Qué hace: Inicia el proceso para crear una nueva app y te conecta con tu cuenta de Google para que AppSheet pueda acceder a tus archivos de Google Sheets.
  • Para qué sirve: Es el primer paso para que AppSheet sepa dónde están tus datos.
  • Verificación: Deberías ver una pantalla de bienvenida de AppSheet o una opción para "Make a new app" (Crear una nueva app) o "Start with your own data" (Empezar con tus propios datos).
Paso 1.7: Elegir "Start with your own data" (Empezar con tus propios datos)
  • Acción: Haz clic en la opción "Start with your own data".
    • Dale un nombre a tu app, por ejemplo: ServinfanciaApp.
    • Elige una categoría, por ejemplo: Operations and Field Service o Education. (Esto es solo para organización).
    • Haz clic en "Choose your data" (Elige tus datos).
  • Qué hace: Le dice a AppSheet que quieres usar un archivo de datos que ya tienes (nuestro Google Sheet).
  • Para qué sirve: Para conectar tu app al archivo DatosAppServinfancia que creamos.
  • Verificación: Se abrirá un selector de archivos de Google Drive.
Paso 1.8: Seleccionar Nuestro Google Sheet
  • Acción: En el selector de archivos, busca y selecciona el archivo DatosAppServinfancia.xlsx (o como lo hayas nombrado). Haz clic en "Select".
  • Qué hace: Le dice a AppSheet exactamente qué archivo de Google Sheets usar.
  • Para qué sirve: AppSheet ahora leerá las hojas de este archivo como tablas para tu app.
  • Verificación: AppSheet empezará a "pensar" y procesar tu archivo. Puede que te muestre una vista previa de tu app con alguna tabla que haya encontrado (si la hoja ConfiguracionApp tenía datos, podría intentar crear una vista para ella). ¡No te preocupes por lo que aparezca aún! Serás redirigido al editor de AppSheet.
Paso 1.9: Configurar la Autenticación (Inicio de Sesión con Google)

AppSheet, por defecto, cuando creas una app e indicas que usarás Google Sheets, suele configurar la autenticación con Google automáticamente si tu cuenta es de Google. Vamos a verificarlo y asegurarnos.

  • Acción: En el editor de AppSheet, busca en el menú de la izquierda la sección Security (Seguridad). Haz clic en ella. Luego, haz clic en Require Sign-In (Requerir inicio de sesión).
  • Qué hace: Te lleva a las configuraciones de seguridad de tu app.
  • Para qué sirve: Aquí nos aseguramos de que la gente necesite una cuenta de Google para usar la app.
  • Verificación:
    • La opción "Require user signin?" (¿Requerir inicio de sesión del usuario?) debería estar Activada (ON).
    • Debajo, en "Authentication provider" (Proveedor de autenticación), debería estar seleccionado "Google" (o el que usaste). Si no, cámbialo a Google.
    • Guarda los cambios si hiciste alguno (suele haber un botón "Save" en la parte superior derecha del editor de AppSheet).

Importante: Al principio, mientras desarrollas, puede que quieras añadir tu propio email como usuario de prueba en la sección Users (Usuarios) dentro de Security, para poder probar la app. Cuando la compartas, podrás gestionar quién más puede usarla.

Paso 1.10: Crear la Pantalla de Bienvenida (Vista)

Ahora vamos a crear esa primera pantalla que verán los usuarios después de iniciar sesión.

  • Acción: En el menú de la izquierda del editor de AppSheet, ve a UX (Experiencia de Usuario) o Views (Vistas).
  • Qué hace: Te muestra las diferentes pantallas (vistas) de tu app y te permite crear nuevas.
  • Para qué sirve: Para diseñar cómo se ve y se navega tu app.
  • Verificación: Verás una lista de las "Primary Views" (Vistas Primarias). Si AppSheet creó alguna vista automáticamente a partir de tu hoja ConfiguracionApp, podrías verla aquí. Podemos borrarla o modificarla.
Paso 1.11: Añadir una Nueva Vista para la Bienvenida
  • Acción:
    1. Si hay vistas existentes que no quieres, puedes pasar el mouse sobre ellas y hacer clic en el ícono de bote de basura para eliminarlas (con cuidado).
    2. Haz clic en + New View (Nueva Vista) o un botón similar para agregar una pantalla.
    3. En el panel de la derecha que aparece para configurar la nueva vista:
      • For this data (Para estos datos): Aquí es donde normalmente seleccionas una tabla. Para una vista de bienvenida simple que no muestra datos de una tabla, puedes seleccionar "None (creates a read-only view)" o, si creaste la hoja ConfiguracionApp y quieres usarla para el mensaje (más avanzado), la seleccionarías aquí. Por ahora, si no hay "None", puedes seleccionar temporalmente ConfiguracionApp (si la creaste y AppSheet la importó como tabla) o cualquier tabla que AppSheet haya podido crear. Lo importante es que el tipo de vista sea "Detail" y la configuremos para que no muestre columnas.
      • View name (Nombre de la vista): Escribe Bienvenida. Este es el nombre que verás en el editor y podría aparecer en el menú si es una vista primaria.
      • View type (Tipo de vista): Selecciona Detail (Detalle). Las vistas de detalle son buenas para mostrar información sobre un solo "algo". En este caso, será nuestra "pantalla de bienvenida".
  • Qué hace: Crea una nueva pantalla en tu app. La vista de "Detail" normalmente muestra los detalles de una fila de una tabla. La usaremos de forma un poco diferente aquí para nuestra bienvenida estática.
  • Para qué sirve: Esta será nuestra pantalla de inicio.
  • Verificación: Deberías ver Bienvenida en tu lista de vistas y una vista previa de la pantalla en el emulador de la derecha. Aún no se verá como queremos.
Paso 1.12: Configurar la Vista de Bienvenida (Título y Mensaje)
  • Acción: Con la vista Bienvenida seleccionada, busca las siguientes opciones en el panel de configuración de la vista (a la derecha):
    • View Options (Opciones de Vista) o Display (Mostrar):
      • Title (Título): Aquí puedes intentar poner "Bienvenidos a Servinfancia". Sin embargo, el "Title" de la vista a veces se comporta más como el nombre de la pestaña. La forma más robusta de poner un gran encabezado es usando una columna (ver paso siguiente si esto no da el resultado deseado).
      • Show column headers (Mostrar encabezados de columna): Asegúrate de que esté Desactivado (OFF), ya que no queremos mostrar nombres de columnas.
      • Column order (Orden de las columnas): Si seleccionaste una tabla (como ConfiguracionApp), esta sección te permite elegir qué columnas mostrar. ¡Queremos que esté vacía o que no muestre ninguna columna real de datos!
    • Para el mensaje principal y el título, una forma común de hacerlo en una vista "Detail" es crear "Virtual Columns" (Columnas Virtuales) en la tabla asociada a la vista (si usaste una) o usar las capacidades de "Show If" de los elementos de la vista.
      Una forma más directa para un mensaje de bienvenida estático usando una vista "Detail" sin datos de tabla es un poco un truco, o mejor, usar una vista tipo "Card" o "Dashboard" vacía con un texto.
      Alternativa más simple para el mensaje estático usando "Detail View" con una tabla de apoyo (como ConfiguracionApp):
      1. Asegúrate que tu hoja ConfiguracionApp tenga al menos una fila y dos columnas, por ejemplo:
        ID_DummyMensaje_Pantalla
        1Bienvenidos a Servinfancia
        (Y otra columna para el segundo mensaje si es necesario).
        Importante: Ve a AppSheet, en Data > Tables, selecciona la tabla ConfiguracionApp y haz clic en "Regenerate Structure" para que AppSheet vea las nuevas columnas.
      2. Vuelve a la configuración de tu vista Bienvenida (UX > Views > Bienvenida).
        • Asegúrate que "For this data" sea ConfiguracionApp.
        • En "Column order", añade solo la columna Mensaje_Pantalla.
        • En "Display mode" o "Main image / Header / Text body", busca cómo asignar columnas a estas áreas. Para una vista "Detail", las columnas en "Column order" se muestran una debajo de la otra.
        • Para el título "Bienvenidos a Servinfancia", podrías tenerlo como el primer valor de la primera columna visible.
        • Para el mensaje "Seleccione una opción para gestionar pedidos", podrías usar una columna de tipo "Show" (Mostrar) con la categoría "Text" (Texto) y en su contenido (Content) escribir directamente el mensaje. Para esto:
          1. Ve a Data > Columns y selecciona la tabla ConfiguracionApp.
          2. Añade una Columna Virtual (haz clic en "Add Virtual Column").
            • Column Name: TituloBienvenida
            • App Formula: "Bienvenidos a Servinfancia" (¡con las comillas!)
            • Type: Show
            • En las opciones de la columna "Show", ponle de Category Header o Section Header.
            • En Content o Value para esta columna Show, pon "Bienvenidos a Servinfancia". (A veces la fórmula ya es suficiente).
          3. Añade otra Columna Virtual:
            • Column Name: MensajeSeleccion
            • App Formula: "Seleccione una opción para gestionar pedidos"

Paso 2.4: Configurar los Tipos de Columna para la Tabla "Ninos"

AppSheet es inteligente, pero a veces necesita que le ayudemos a entender qué tipo de información va en cada columna. Esto es CRUCIAL.

  • Acción:
    1. En AppSheet, ve a Data > Columns.
    2. En el desplegable "Select a table", elige la tabla Ninos.
    3. Verás la lista de columnas que creaste. Revisa y ajusta la columna "TYPE" (Tipo) para cada una:
      • ID_Nino: Tipo Text. Marca la casilla "KEY?" (¿Es llave?). Esto le dice a AppSheet que esta columna tiene un valor único para cada niño, ¡súper importante!
        • Opcional: En "Initial value" (Valor inicial) puedes poner la fórmula UNIQUEID(). Esto hará que AppSheet genere un código único cada vez que agregues un nuevo niño.
      • NombreCompleto: Tipo Name (Nombre) o Text.
      • FechaNacimiento: Tipo Date (Fecha).
      • EdadMeses: Tipo Number (Número).
        • Aquí pondremos una fórmula para que se calcule sola. Haz clic en el lápiz para editar esta columna.
        • Busca el campo "Formula" o "App Formula".
        • Escribe la siguiente fórmula (¡con cuidado!):
          FLOOR((TODAY() - [FechaNacimiento]) / 30.4375)
          Explicación de la fórmula: TODAY() es la fecha de hoy. Le restamos la [FechaNacimiento], lo que da el número de días. Dividimos por 30.4375 (promedio de días en un mes) para obtener meses. FLOOR() redondea hacia abajo al número entero más cercano.
        • Marca la casilla "Editable?" (¿Editable?) para esta columna como DESMARCADA (o busca "Editable if" y pon la condición FALSE). No queremos que la gente edite esto manualmente.
      • GrupoEtarioActual: Tipo Text (o podrías usar Enum si tienes una lista fija de grupos etarios. Por ahora, Text está bien).
        • También le pondremos una fórmula. Edita la columna.
        • En "App Formula", podrías empezar con algo simple o dejarlo para una acción de actualización. Para que se calcule automáticamente basado en EdadMeses (simplificado, necesitarás definir tus rangos): IFS( AND([EdadMeses] >= 0, [EdadMeses] <= 5), "0-5 Meses", AND([EdadMeses] >= 6, [EdadMeses] <= 11), "6-11 Meses", AND([EdadMeses] >= 12, [EdadMeses] <= 23), "12-23 Meses (1 año)", AND([EdadMeses] >= 24, [EdadMeses] <= 35), "24-35 Meses (2 años)", AND([EdadMeses] >= 36, [EdadMeses] <= 47), "36-47 Meses (3 años)", AND([EdadMeses] >= 48, [EdadMeses] <= 59), "48-59 Meses (4 años)", [EdadMeses] >= 60, "5 años o más", TRUE, "Revisar Edad" )
          ¡Ajusta estos rangos y nombres de grupo según los que usa Servinfancia! IFS() revisa condiciones en orden. La última condición TRUE, "Revisar Edad" es por si algo no encaja.
        • También desmarca "Editable?".
      • FechaUltimaActualizacionGE: Tipo DateTime (Fecha y Hora) o Date.
        • Si quieres que se actualice cada vez que el Grupo Etario cambie, puedes usar una Acción (lo veremos después) o en la configuración de la columna GrupoEtarioActual, en la sección "Behavior" (Comportamiento) o "Updates" (Actualizaciones), podrías configurar "Reset on edit?" para la columna FechaUltimaActualizacionGE para que se actualice con NOW() cuando GrupoEtarioActual cambie. Esto es un poco más avanzado.
          Una forma más simple es que esta fecha se llene cuando se guarda el registro. En "Initial Value" o "App Formula" para FechaUltimaActualizacionGE, podrías poner NOW(), pero esto se actualizaría cada vez que se sincronice la fila. Una mejor manera es usar una Acción. Por ahora, la dejamos así y la llenaremos con una Acción más adelante o al guardar.
      • Responsable: Tipo Email.
        • En "Initial Value", puedes poner USEREMAIL() para que se llene automáticamente con el email del usuario que está agregando o editando el registro.
      • Notas: Tipo LongText (Texto Largo).
    4. Revisa la columna "LABEL?" (¿Etiqueta?). Usualmente se marca para la columna que mejor describe la fila, como NombreCompleto. Esto es lo que AppSheet mostrará en listas desplegables cuando se refiera a un niño.
    5. ¡No olvides hacer clic en Done (Hecho) después de editar cada columna y luego Save (Guardar) los cambios generales de la app!
  • Qué hace: Le dice a AppSheet qué tipo de datos contiene cada columna y cómo debe comportarse (ej. si es una llave, si se calcula sola, si es editable). Esto es fundamental para que la app funcione bien.
  • Para qué sirve: Para que los formularios de entrada de datos pidan el tipo correcto de información (ej. un calendario para fechas), para que las fórmulas funcionen, y para que los datos se muestren correctamente.
